# 05. 機能要件

## 5.1 基本表示
- 16 進表示と ASCII 表示の 2 カラム表示、行頭に 64bit オフセットを表示
- 1 行 16 バイト（カスタマイズ可能）、現在バイトをハイライト

## 5.2 ナビゲーション
- オフセット直接指定 (`goto 0x1A3F`)
- 相対移動（±N バイト、ページ単位、ファイル先頭/末尾）
- ブックマーク（ラベル付き最大 32 件）

## 5.3 編集
- 単一バイトまたは選択範囲の書き換え（16 進/ASCII 双方入力）
- 連番/パターン入力（例: `fill 0xFF count=32`）
- 差分プレビュー（保存前の変更確認）
- Undo/Redo（最小 128 ステップ、リングバッファ）

## 5.4 検索・置換
- 16 進/ASCII/正規表現（ASCII 部分のみ）検索
- 検索結果リストとジャンプ
- 選択範囲に限定した置換

## 5.5 ファイル入出力
- 読み取り専用モード
- 自動バックアップ（`.bak`）と復元
- 変更部分のみをパッチファイル（diff フォーマット）として出力
- 標準入力/標準出力へのストリーム対応

## 5.6 CLI/TUI
- `hex-editor view <file>`: 読み取り専用 TUI
- `hex-editor edit <file>`: 編集 TUI
- `hex-editor patch`: CLI ワンショット編集
- `hex-editor diff <fileA> <fileB>`: 16 進差分
